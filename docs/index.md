# Compute Central

Compute Central is a practical documentation hub for DevOps, cloud engineering, and SRE work. The focus is on clear guides, useful examples, and operational workflows you can actually apply.

## What You Will Find Here

- Infrastructure and automation guides
- Kubernetes and container learning material
- CI/CD, quality, and observability notes
- Interview preparation and system design notes
- Hands-on examples that stay close to real work

## What Makes the Docs Different

The goal is not to collect theory for its own sake. These docs are written to help you:

- Understand the topic quickly
- Try the commands with confidence
- Connect one tool to the next in a real workflow

## Recommended Reading Flow

1. Start with the overview page for a topic.
2. Move to the setup or fundamentals page.
3. Use the examples and verification commands to test what you learned.
4. Return to troubleshooting pages when something breaks.

## About the Project

The site brings together practical notes and patterns from real infrastructure learning and hands-on platform work. The aim is simple: make systems more reliable, automated, observable, and easier to operate.

## About Sameer Alam

Sameer Alam is a DevOps engineer and SRE-focused practitioner who works on infrastructure automation, platform reliability, observability, CI/CD, and practical cloud operations.

## What This Site Represents

Compute Central is a long-running learning and writing project that started in 2016. It began as a way to document technical learning and grew into a structured space for practical engineering notes.

## Focus Areas

- DevOps and platform engineering
- Site reliability and operations
- Kubernetes and containers
- CI/CD and automation
- Monitoring, logging, and troubleshooting
- Applied AI for engineering workflows

## Writing Approach

The goal of the site is to make technical topics:

- Simpler to understand

- More useful in real operational work
